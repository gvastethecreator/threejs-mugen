# Playable v0 Status

This document is the working truth for the private sandbox milestone. It separates the playable local prototype from real MUGEN compatibility.

## Closed In The Current Playable Prototype

- Vite, TypeScript, Three.js local app.
- First screen opens in Runtime Mode.
- Three active local original fighters are selectable: Nova Boxer, Mira Volt, Rook Apprentice.
- The local runtime fighters load atlas textures from `public/characters/*/sprite-sheet-alpha.png` and `manifest.json.frame_layout`.
- Nova Boxer, Mira Volt, and Rook Apprentice include freshly generated imagegen sprite-sheet sources, normalized atlas frames, QA contact sheets/GIFs, `sprite-atlas-builder` contracts, and MUGEN-lite `.def/.air/.cmd/.cns` templates under `public/characters/<fighter>/mugen/`.
- The three active fighters use 8-frame, slow-cadence `walk-forward` rows generated as new sprites, not mechanical repairs. The current rows were replaced from fresh one-row imagegen outputs, recomposed into the atlas, and checked with `sprite-atlas-builder/scripts/check_motion_variation.py --states walk-forward`. The heuristic returns `ok: true` with one conservative low-body-bob warning per fighter, so the runtime surfaces `walk QA warn 1` and the contact sheets/browser remain the final visual check.
- Crouch and jump scale are now controlled by character-level atlas normalization instead of per-frame fitting. Browser screenshots verify crouch as `anim 10`/`stateType C` and jump as `anim 40`/`stateType A` without the previous oversized pose effect.
- Runtime Mode now shows those local atlas motion reports as `walk QA` badges in the roster, HUD, compatibility panel, exported JSON, and browser diagnostics through `window.__MUGEN_WEB_SANDBOX__.runtimeRoster` / `.atlasMotionQa`.
- Rooftop Dojo is the default original native stage, with imagegen-generated PNG art, floor, bounds, parallax layer, camera, grid, and axis.
- Training Grid remains available as a geometry-only fallback stage.
- P1 can walk, crouch, jump, punch, and kick.
- CPU P2 pressures, walks into range, and attacks.
- Combat has hurtboxes, active hitboxes, hit pause, hit stun, pushback, life damage, and power gain.
- Runtime has stage HUD, life/power bars, timer, fight/KO/time-over message, pause, step, speed, reset, and mobile touch controls.

- Inspector loads ZIP/folder characters and parses DEF/AIR/CMD/CNS/ST enough to inspect.
