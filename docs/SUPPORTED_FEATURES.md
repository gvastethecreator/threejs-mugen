# Supported Features

## Loader

- ZIP packages.
- Browser folder input.
- `.def` discovery in root or nested character folder.
- Case-insensitive virtual path resolution.
- UTF-8/BOM text decoding with Windows-1252 fallback.
- Optional `data/fight.def` discovery for system presentation assets. `[Files] fightfx.air` and `fightfx.sff` are resolved relative to `fight.def`, with basename fallback to `fightfx.air` / `fightfx.sff` when refs are omitted. Parsed FightFX AIR actions are exposed as both `fightfx` and current common/default hit-spark libraries, and decoded FightFX SFF sprites are registered through a global `SffSpriteProvider` route when available. This is first-pass system spark asset plumbing, not full motif/screenpack parsing or exact MUGEN system-asset ownership.

## DEF

- `[Info]`: `name`, `displayname`, `versiondate`, `mugenversion`, `author`.
- `[Files]`: `cmd`, `cns`, `st`, `stcommon`, `sprite`, `anim`, `sound`, `pal1` through `pal12`.
- Raw sections and diagnostics.

## AIR

- `[Begin Action N]`.
- Frames: `group, image, x, y, time`, optional `flip`, optional `blend`.
- `Loopstart`.
- `Clsn1`, `Clsn2`, `Clsn1Default`, `Clsn2Default`.

## SFF

- Header/version detection.
- Initial SFF v1 subfile parsing.
- Initial 8-bit, single-plane PCX RLE decoding.
- SFF v1 palette propagation for normal and previous-palette sprites.
- SFF v1 `samePalette` PCX subfiles without embedded palette blocks decode against the previous palette instead of truncating the RLE stream.
- Linked SFF v1 sprite records.
- Decoded SFF v1 sprites are exposed through `SffSpriteProvider` and can render in Inspector Mode.
- SFF v2 header, sprite table, palette table, format counts, and palette records.
- SFF v2 RAW8/RAW24/RAW32 decoding.
- SFF v2 RLE8 indexed-sprite decoding.
- SFF v2 RLE5 and LZ5 5-bit indexed-sprite decoding.
- SFF v2 unsupported format counts are surfaced in diagnostics and compatibility reports.
- Unsupported or undecodable sprites fall back to mock sprites through `SpriteProvider` so inspection can continue.

## Stage DEF

- Stage `.def` discovery from ZIP/folder packages when files live under `stages/`.
- `[Info]`, `[Camera]`, `[PlayerInfo]`, `[StageInfo]`, `[Music]`, `[BGDef]`, and `[BG ...]` section parsing.
- Imported stage runtime mapping for display name, side bounds, camera start/zoom, player starts/facing, zoffset, and localcoord.
- Stage sprite/music path resolution from the stage folder, package root, and `data/` style fallbacks.
- Parsed `[BG ...]` sections preserve `id`, `spriteno`, `actionno`, `start`, `delta`, `layerno`, `tile`, and `tilespacing` metadata.
- `[BGCtrlDef ...]` and `[BGCtrl ...]` sections are parsed into Stage BG controller IR with group name, inherited `looptime`, inherited/per-controller `ctrlID`, controller type, timing window, params, and target layer ids.
- Recognized BGCtrl types are `Null`, `Visible`, `Enabled`, `VelSet`, `VelAdd`, `PosSet`, `PosAdd`, `Anim`, `SinX`, and `SinY`. They execute through a bounded renderer path for visibility/enabled skips, position/velocity offsets, action swaps, and simple sine offsets. Exact MUGEN timing, side effects, window/mask/trans behavior, and full parity remain unsupported.
- Static normal BG sprites can render from the stage SFF when the referenced `spriteno` is decoded by the existing SFF reader.
- Embedded stage `[Begin Action N]` blocks are parsed for action-backed BG layers.
- Action-backed BG layers can render when their referenced AIR frames resolve to decoded stage SFF sprites.
- Basic horizontal camera parallax is applied from layer `delta`.
- Basic bounded sprite tiling is applied from `tile`/`tilespacing` so repeated floor/wall layers can fill the viewport.
- Stage compatibility summaries report DEF/SFF/music presence, decoded stage sprites, BG sprite coverage, animated BG coverage, tiled layers, bounded BGCtrl coverage, placeholder fallbacks, warnings, and known unsupported stage features.
- Stage compatibility reports now include a first per-layer BG IR: each imported `[BG ...]` layer records its source section, type, layer number, control id, start, delta, tiling, sprite/action reference, decoded-frame coverage, unsupported layer features, fallback reason, and status as `rendered`, `animated`, `fallback`, `missing`, or `unsupported`.
- Stage compatibility reports include BG controller rows with bounded/unsupported status, timing, ctrl ids, resolved target layer labels, raw params, unsupported type notes, and fallback notes for exact parity gaps.
- Placeholder runtime background bands remain as a fallback when a stage BG sprite is missing or not decodable.

## CMD

- `[Command]` blocks with `name`, `command`, and `time`.
- `[Defaults]` `command.time`, partial `command.steptime`, `command.buffer.time`, and partial `command.buffer.hitpause`.
- Per-command `time`, partial `steptime`, `buffer.time`, and partial `buffer.hitpause` overrides.
- `[Remap]` button aliases are parsed before command tokenization; remapped commands show their original and resolved sequences in the UI, and blank remap targets disable the affected command path.
- Direction/button/modifier/separator/combo token classification.
- `CommandBuffer` stores inputs by frame, exposes a bounded debug history of recent samples, and can query active commands with duplicate-name alternatives.
- Hit-pause input samples can be buffered for commands that opt into `buffer.hitpause`; hit-pause samples are ignored for commands that opt out.
- Derived diagonals (`DF`, `DB`, `UF`, `UB`) from simultaneous cardinal input.
- Command matching collapses input into an 8-way direction so `DF` does not satisfy a plain `F` step unless the command uses `$F`.
- Repeated direction/button steps such as `F, F` require a fresh activation instead of matching a held input across adjacent frames.
- Plain direction/button inputs now require a fresh press/change within the buffer window instead of staying active for the full duration of a hold.
- Partial `/` hold matching for buttons and direction families, such as `/x` and `/$D`.
- Partial `~` release matching for buttons and directions, including common direction-release transitions such as `~D, DF`.
- Partial numeric charge matching on `/N` and `~N` modifiers, including command forms such as `~30$D, F, x`.
- Partial `steptime` matching rejects commands when consecutive matched steps are farther apart than the command's step window; when unset it falls back to the command's total `time`.
- `$` direction-family matching for commands such as `/$D`.
- Same-frame `+` button combos.
- Same-step `|` alternatives for directions/buttons, including alternatives inside a same-frame `+` combo such as `x+y|z`.
