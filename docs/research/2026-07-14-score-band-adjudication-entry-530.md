# Research: Score-band adjudication, Entry 530

## Question

Does the evidence now satisfy the written entry criterion for the next
practical MUGEN compatibility band, or is the project still correctly capped
at 35/100?

## Criteria

The scorecard defines the `36-55` practical MVP band as an official/local
KFM-style fixture executing common authored routes with visible reports. The
claim ledger keeps that wording at `Executed Partial`; it does not permit
`KFM works`, full compatibility, or parity wording without oracle comparison.

## Evidence reviewed

- Optional local official KFM artifacts are present and passed for basic
  movement, normal attack, QCF special, guard, hitstun, get-hit, fall, and
  ground/air recovery routes.
- The repository-owned CC0 MUGEN-lite journey crosses DEF/CMD/CNS/AIR/SFF,
  deterministic ZIP transport, runtime traces, browser evidence, and visible
  unsupported findings.
- The official Training Room stage route passes production ZIP import, Studio
  Stage selection, StageCompatibilityReport output, desktop/mobile render
  proof, and native closeout. It is classified `optional-private`.
- `CompatibilityCorpus/v0` now indexes character and stage journey schemas,
  keeps unsupported density and provenance references, and rejects invalid
  journey discriminators.
- Native closeout is green at 211 test files / 2129 tests, TypeScript 7,
  build, boundaries, CSS budget, and 600/600 trace artifacts.

## Decision

Promote **Practical MUGEN compatibility: 35 -> 36 / 100**, the minimum point
inside the written `36-55` band. This is a threshold decision, not a claim of
breadth: the route proves a bounded KFM-style execution/report path, while the
score remains far from the 45+ milestone target and from the 56-75 broad-subset
band.

Leave these scores unchanged: playable private sandbox `65`, MUGEN 1.0/1.1
MVP `20`, full MUGEN `10-12`, IKEMEN `6-8`, and Creator Studio `25`.

## Adversarial check

The strongest objection is that the official KFM package is a local optional
fixture and the stage sample is noncommercial. The decision answers this by
using the exact local-fixture wording, keeping the stage optional-private,
retaining all blocked claims, and moving only the practical score's first band
point. Public corpus breadth, commercial redistribution, and exact parity are
not counted.

## Next

Select the next source-backed stage/runtime gap, prioritizing BGCtrl timing,
stage camera/window/mask behavior, or another independently evidenced package
route. Do not infer another score movement from this decision alone.
