# Fixture Goldens

This document defines the fixture and golden-artifact strategy. It keeps fixture-backed compatibility claims separate from parser counts and generated/native runtime claims.

## Fixture Classes

| Class | Examples | Role | Required In Repo |
| --- | --- | --- | --- |
| Native generated | Nova Boxer, Mira Volt, Rook Apprentice | Always-playable local baseline. | Yes, authored/generated assets are included. |
| Synthetic imported | Small in-test characters/states | Portable controller/runtime gates. | Yes, generated by tests/scripts. |
| Official optional | KFM, KFM720 from Elecbyte package | Main real MUGEN fixture gates. | No, local `.scratch` only. |
| Third-party optional | CodeFuMan, SF3 Ryu demo, other packs | Stress parser/report/runtime paths. | No, local `.scratch` only. |
| Imported stage optional | KFM stage, stage0-720, others | Stage parser/render diagnostics. | No, local `.scratch` only. |

External fixtures must remain outside committed source unless they are original/local generated assets.

## Required Trace Artifacts

Portable required artifacts live under:

```txt
.scratch/qa/trace-gates/
```

Required artifacts should prove engine plumbing without private assets:

- native hit and whiff routes
- synthetic imported State -1 command route
- reject/guard/hitstun/fall metadata
- custom get-hit controller-flow
- default Common1 get-hit and progression
- synthetic bounded fall recovery
- synthetic bounded recovery input
- synthetic bounded recovery threshold
- synthetic bounded recovery actor-frame tick order
- synthetic bounded air recovery velocity
- synthetic bounded ground recovery selection/velocity
- synthetic bounded round KO/time-over snapshot evidence
- state exit
- Target*
- Pause/SuperPause
- Projectile
- Helper
- Explod
- ModifyExplod live-mutation
- HitDef hit/guard sound and spark routes, including combined hit sound + FightFX hit-spark and combined guard sound + FightFX guard-spark package routes

These artifacts can prove regression stability and controller-system behavior. They do not prove official character parity.

## Optional Official Artifacts

Optional artifacts run only when the matching local fixture exists:

```txt
.scratch/fixtures/kfm-official.zip
.scratch/fixtures/kfm720-official.zip
```

Optional KFM artifacts may prove:

- `x -> 200`
- `QCF_x -> 1000`
- guard route
- hitstun/get-hit entry
- state exit
- Common1 state `5000`
- stand progression `5000 -> 5001 -> 0`
- fall/ground-impact/bounce/lie-down entry `5000 -> 5030 -> 5050 -> 5100 -> 5101 -> 5110`
- lie-down/get-up recovery completion `5110 -> 5120 -> 0`
- air recovery-input completion `5050 -> 5210 -> 52 -> 0`
- early recovery-input rejection while still in `5050`
- ground recovery-input completion `5050 -> 5200 -> 5201 -> 52 -> 0`

Skipped optional artifacts are not passes.

## Golden Artifact Fields

Each artifact should include:

- artifact schema/version
- target id and label
- fixture/source type
- input script frames
- final actor/effect summaries
- checksum
- routed states
- executed states
- executed controllers
- executed typed operations
- combat reasons/events
- gates with pass/fail evidence
- notes and explicit remaining gaps

## Updating Goldens

Only update golden expectations when:

- behavior change is intentional
- docs/status explain the change
- focused tests pass
- `pnpm qa:trace` passes
- visual QA runs if visible behavior changed

Do not update checksums just to make a failing trace pass.

## Claim Map

| Claim | Minimum Proof |
| --- | --- |
| Native roster playable | `qa:smoke` runtime screenshots plus native trace gates. |
| Imported route executes | Synthetic trace with routed state, executed controller, and final actor evidence. |
| Official KFM route executes | Optional KFM trace artifact with fixture loaded and relevant states/controllers. |
| Controller family partial support | Synthetic trace requiring typed operation or explicit controller execution. |
| Fixture parity | Official fixture artifact plus known residual gaps; currently not claimed. |
| Stage render support | Stage report plus browser screenshot. |
| IKEMEN support | Scanner report only until execution artifacts exist. |

## Negative Fixtures

Add negative tests/artifacts for:

- malformed DEF paths
- missing `common1.cns`
- unsupported SFF encodings
- unknown controllers/triggers
- IKEMEN-only ZSS/Lua files
- stage BGCtrl/window/mask/parallax features
- missing sprites/animations referenced by CNS/AIR

Failures should report unsupported/unknown features instead of crashing.
